Installing WebScarab

The latest stable release of WebScarab is always available from the OWASP download page at SourceForge.net. Sometimes, an unstable testing release may be posted at this site for broader testing.

Previous releases of WebScarab were distributed as an installer, which, when run, extracted the necessary files to a directory, and on Windows platforms, created shortcuts and menu entries. In an effort to reduce complexity (and the download size!), WebScarab is now distributed as an entirely self-contained jar, which needs no installation at all.

One downside of this new distribution method is that the source code is no longer included in the distributed file. If there is demand for it, I will make source tarballs available for download at the same time as the compiled executable. Let me know if this is an issue for you. Otherwise, the source can always be retrieved from the SourceForge CVS servers.

If you intend to modify the sources and rebuild, it is not recommended to check them out into a directory that contains spaces, as the build system (ant) does not handle this very well. On win32, I suggest using a directory immediately off the root, e.g. C:\WebScarab.
