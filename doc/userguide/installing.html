Installing WebScarab

The latest version of WebScarab is always available from the OWASP download page at SourceForge.net WebScarab comes packaged in a self-executing .jar that can install WebScarab into the appropriate places. The installer includes some now-obsolete design documentation, the source that was used to build WebScarab, as well as most library dependencies that WebScarab requires to run.

The installer will try to install WebScarab into the "Program Files\WebScarab" directory on a Win32 platform, or into your home directory on a Unix platform. If you intend to modify the source and rebuild, it is not recommended to install into a directory that contains spaces, as the build system (ant) does not handle this very well. On win32, I suggest installing into a directory immediately off the root, e.g. C:\WebScarab.
