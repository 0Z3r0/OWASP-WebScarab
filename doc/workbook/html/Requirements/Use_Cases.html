User Cases

  • Configure Tool

    • Configure Main Options - User is presented with a main configuration panel

      1. Set modules path - user enters the path to the modules root directory

      2. Set VulnXML path - user enters path to the VulnXML directory

    • Configure Spider - User is presented with the spider configuration panel

      1. Set spider style - breadth first or depth first

      2. Set Threads - sets no of threads for spider

      3. Set Page Time-outs - sets a time-out value

      4. Set 404's - sets custom 404 pages the application may use

      5. Set Honor Robot Exclusions - Boolean

      6. Set target HTTP port

    • Configure Proxy Recorder - User is presented with the proxy configuration panel

      1. Set proxy port

      2. Start / stop / pause proxy ?

      3. Set target HTTP port.

    • Configure DataBase - User is presented with the database configuration panel

      1. Set db type - User sets db type such as Oracle, MySQL etc

      2. Set Connection method - user sets connection method such as username / password

      3. Test Connection - user clicks button to test db connection - Boolean

    • Configure Analysis Engine

    • ? Not sure - maybe threading only

    • Load Modules - User is presented with the Modules Main Screen

      1. Check for new modules - user clicks button and checks to see if a later modules list file is available

      2. Load new modules - user loads new test modules from local class files.

    • Configure Specific Module

      1. Load specific module config panel - user selects a specific module config

      2. Sets module config - user selects options and saves on exit

    • Load VulnXML

      1. Check for new files - user clicks button and checks if a later file(s) is available

      2. Load new checks - user loads new checks from local XML files.

    • Help

      1. Get version - user selects version menu item and is returned the build number

      2. Get API Documentation - user selects menu item and tool spawns local javadoc in default browser.

  • Analyze Site

    • New Session

      1. Create a new session - choose a session identifier to group settings and results into.

      2. Set application discovery location scope - user sets the scope of the application discovery. Options would include site root, excluded path, not spider outside of primary DNS domain, authentication credentials.

    • Spider or Proxy Recorder

        1. Spider - start / stop and pause spider to crawl the site and discover applications that can be tested

          • Enters Auth credentials when prompted - if the spider asks the user for auth credentials, the user enters them into a pop-up box and presses continue

          • When prompted for wide scoped form (say form with limitless inputs) user prompted to skip / enter value range or specific values. User enters values and continues.

        2. Start proxy recorder - use regular web browser (Internet Explorer or Mozilla etc) to browse and interact with web applications.

        3. Stop proxy recorder - click button to end proxy recording session

    • Generate Security Tests

      1. Generate tests - user clicks button to generate the tests for the session data from the spider proxy

      2. Validate tests - user approves or rejects the proposed tests being built

    • Execute Security Tests

      1. Select session - user selects session ID tests to execute

      2. Go - User clicks button and execute tests

      3. Pause Tests - user clicks pause button to pause tests

      4. Stop Tests - User clicks button to terminate tests.

    • Generate Reports

  1. Select Session

  2. Select report type

  3. Generate report
